# 开发日志 2026-06-12

## 已完成

### pop_card 功能

- 初始发牌后，自动将手牌中最右边第一张牌 pop 出，作为 active card。
- 流程：发牌完成 → 取最右侧牌 → 弹出到 ActiveCardSlot。

## 待解决问题

### 1. 激活牌插入位置检测有误

- **现象：** 将 active card 拖入手中时，有时无法正确插入到目标位置。
- **影响：** 牌可能插入到错误的空隙，或插入失败。
- **下次计划：** 排查 `HandCardQueue` 中的 slot/gap 位置计算逻辑，确保拖拽落点与插入索引的映射正确。

### 2. 同花色打出后左侧牌右移的碰撞检测未实现

- **现象：** 同花色牌打出形成组合后，左侧的牌应该自动右移填补空位，但目前碰撞检测逻辑尚未处理此情况。
- **影响：** 手牌队列在移除匹配牌组后不会自动重新排列。
- **下次计划：** 与问题 1 集中解决，重构 `HandCardQueue` 的布局更新与碰撞检测逻辑。
